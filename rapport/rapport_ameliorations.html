Persistance des commandes en base de données
CRITIQUE

Les commandes n'étaient pas sauvegardées — elles disparaissaient après le paiement. Le jury ne pouvait pas vérifier qu'une commande avait été passée.

Entité Order avec tous les champs : email, nom, adresse, ville, pays, total, frais de port, statut, articles (JSON), date
Route POST /api/orders — accessible publiquement pour enregistrer une commande
Route GET /api/admin/orders — liste toutes les commandes (ROLE_ADMIN)
Route PATCH /api/admin/orders/{id}/status — changer le statut d'une commande
Commande.jsx — handlePaiement() appelle désormais POST /api/orders avant de vider le panier
// Commande.jsx — AVANT
function handlePaiement(e) {
  e.preventDefault();
  clearCart(); // panier vidé, commande PERDUE
  setEtape(3);
}

// Commande.jsx — APRÈS
async function handlePaiement(e) {
  e.preventDefault();
  await api.post('/api/orders', { email, firstName, items, total, ... });
  clearCart(); // commande SAUVEGARDÉE en BDD ✓
  setEtape(3);
}
